## Background

Once the accounting feature is enabled for a company, the Client must complete the core account setup before using the accounting module. On the client side, the only value the client sets during initial setup is the base currency. The fiscal year-end is set by the accountant on the admin app and is view-only for the client; lock periods are also handled on the admin app. All eligible users get the same (uniform) accounting access — there are no per-user accounting roles on the client app.

## Goal

Allow an authorized user on the client app to complete the accounting initial setup (base currency), view the accountant-set fiscal year, validate the setup, and on success make the saved value available across the client and admin apps.

---

## Acceptance Criteria

1. **Access the initial setup**

GIVEN the accounting feature has been enabled for the company
WHEN an authorized user opens the accounting module before setup is complete
THEN the system presents the accounting initial setup screen
AND access to complete setup is limited to an authorized user

2. **Set base currency**

GIVEN the user is completing account setup
WHEN an authorized user selects a supported base currency (11 currencies available on BA)
THEN the system must display the selected base currency for that company
AND the value is non-editable on the client app once saved
AND the value is editable on the admin app and should be reflected in the next fiscal year

**NOTE - Add a Tip to help client choose the Base Currency**

**How to choose it?**
**Your base currency is used for your annual accounts and financial statements.**
**Choose the currency that best reflects how your business earns revenue and pays major expenses.**
**Once selected, it cannot be changed.**

3. **Fiscal year is view-only**

GIVEN the fiscal year-end is set by the accountant on the admin app
WHEN the client views the accounting setup / settings
THEN the fiscal year is displayed read-only
AND the client cannot set or edit it

4. **Select to Continue and field validations**

GIVEN the user is completing account setup
WHEN the user selects to Continue on the initial setup screen
THEN the system must validate all the setup fields according to the Account Setup Validation Rules table below

5. **Accounting initial setup success (passes validations)**

GIVEN the input on the accounting initial setup screen passed all validations
WHEN the user selected to Continue
THEN the system uses the existing message format to display that the setup is successful
AND the saved value is shown in the account settings on the client app
AND the saved value is reflected in the admin app

6. **Accounting initial setup failed (does not pass validations)**

GIVEN the input on the accounting initial setup screen did not pass all validations
WHEN the user selected to Continue
THEN the system uses the existing message format to display that the setup is not successful
AND displays an in-line error for the information that did not pass validation
AND the system does not save the input value

## Notes

* Client app (BA portal). Initial setup is a one-time step after the accounting feature is enabled.
* Base currency is the only value the client sets; the fiscal year-end is set by the accountant (admin app) and shown read-only to the client.
* Lock periods are configured on the admin app, not the client app.
* Base currency is editable only on the admin app; changes take effect from the next fiscal year.
* Uses the existing message format for success/failure messages.
