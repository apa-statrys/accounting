## Background

For reconciliation, the client brings in transactions from their non-Statrys bank accounts by importing statements. The client first registers an external account (name, bank, currency), then imports its statements by CSV or PDF (OCR). Imported transactions populate that account's Other Bank Accounts transaction history, where the client can then attach supporting documents.

## Goal

Let the client **register** an external bank account and import its statements (CSV or PDF/OCR), with duplicate detection, so the transactions appear in the Other Bank Accounts history ready for reconciliation.

---

## Acceptance Criteria

1. **Register an external account**

GIVEN the client wants to import statements from a non-Statrys account
WHEN they register the external account
THEN they provide the account name, bank, and currency
AND the account becomes available to import statements into

2. **Import a statement (CSV or PDF)**

GIVEN a registered external account
WHEN the client imports a statement
THEN they can upload a CSV or a PDF statement (see Import Methods table)
AND only the accepted file types and size are allowed

3. **System parses and validates the file**

GIVEN a statement has been uploaded
WHEN the system parses it (CSV columns mapped, or PDF read via OCR) and validates it
THEN if the file is invalid it shows an error and asks the client to re-upload
AND if valid the transactions are extracted

4. **Duplicate detection**

GIVEN the imported statement overlaps a previously imported period
WHEN the system detects likely duplicate transactions
THEN it warns the client and lets them choose to import or skip the duplicates

5. **Imported transactions appear in the Other Bank Accounts history**

GIVEN a successful import
WHEN processing completes
THEN the transactions appear in that account's Other Bank Accounts transaction history (Client Story C1)
AND each defaults to Missing Attachment status

## Rules

**Import Methods**

| Method | Behaviour |
| --- | --- |
| CSV upload | Transactions parsed from CSV columns |
| PDF statement | Transactions read via OCR |

**External Account Registration**

| Field | Required |
| --- | --- |
| Account name | Yes |
| Bank | Yes |
| Currency | Yes |

**Duplicate Handling**

| Condition | Behaviour |
| --- | --- |
| Overlapping / duplicate transactions detected | Warn the client; client chooses to import or skip |
| No duplicates | Import all |

---

## Notes

* Client platform only. Statrys BA transactions are already in the system and are not imported here.
* Imported transactions feed the Other Bank Accounts history and default to Missing Attachment.
* Other-account data comes from imported statements only — no live bank feeds.
