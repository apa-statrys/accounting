## Goal
Define every field on a credit note, its source, data type, and editability by status, so the client
edits each CN correctly before confirming and the system enforces the right restrictions per lifecycle
stage.

## Acceptance Criteria
1. **System-generated fields** are pre-filled and read-only, and cannot be edited at any stage.
2. **Pre-filled fields** are populated automatically, editable **before** Create Credit Note, read-only after.
3. **Client-filled fields**: client fills them; can **add / edit / remove line items**; Qty/Unit Price
   changes reflect immediately in Amount/Subtotal/Total; **≥1 line item required** before Create.
4. **After Create (Applied status)**: no field editable, no line item add/edit/remove — only view, send,
   download, cancel.
5. **Discount** optional; entering it recalculates Total Discount + Total; if none, Total Discount hidden
   and Total = Subtotal.

## Field tables (current description — authoritative)
**System-generated (read-only always):** CN number `CN-YYYY-######`, CN type (`Sales credit note`),
CN status (Draft / Applied to [inv] / Cancelled), CN currency (locked to invoice), Company Info, Company Logo.

**Pre-filled (editable before Create, locked after):**
Customer name · Associated invoice number(s) · CN date (DD/MM/YYYY) · **Payment method** (dropdown:
Statrys Primary VA / Other Statrys VA / Other registered bank accounts — default Statrys Primary VA) ·
Client information · Customer address / contact info.

**Client-filled (locked after Create):**
Reason (enum dropdown **+ optional free text**) — Return / Defect / Pricing error / Goodwill / Dispute ·
Allocation per invoice (multi-invoice — deferred, see DES-719) · **Discount** (optional).

**Line item fields (≥1 required, locked after Create):**
Item Name (req) · **Description (req)** · Quantity (req, ≤999,999.99) · Unit (opt: unit/hour/day/month/
session/pair) · Unit Price (req) · Amount (calc = Qty × Unit Price, read-only).

**Editability by status:** Draft = pre-filled/client-filled/line items editable, system + summary read-only.
Applied to [inv] = everything read-only. Cancelled = everything read-only.
