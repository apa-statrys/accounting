# DES-818 — View Credit Notes List

**Status:** To Do · **Fetched:** 2026-07-13 (ticket updated 2026-07-10) · Split out of DES-763.
**Golive:** credit notes ship OFF on prod at launch.

The central register for all credit notes: columns, filters, sorting, per-status actions, and a CN badge
on linked invoices. **Uses the collapsed 3-status model (Draft / Applied / Cancelled)** — same as DES-719.

## Acceptance Criteria
1. **View all** — all CNs shown with the Columns below; default sort Issue Date newest-first; each row → CN detail.
2. **Filter & sort** — filter/sort combine; **active filters shown as removable chips**.
3. **Actions** — only the actions for that CN's current status (see table).
4. **CN badge** — linked invoices show a CN badge in the Sales Invoice List; tapping it → the CN detail view.

## Columns
CN number (`CN-YYYY-######`, links to detail) · **Issue date (DD/MM/YYYY)** · Customer · Related invoice(s)
(links) · Total amount (2 dp, CN currency) · Status.

## Filters
Customer (search by name) · **Status: All (default) / Draft / Applied / Cancelled** · **Date range** (custom, on Issue date).

## Sort
Issue date (newest, default) · CN number (descending) · Total amount (highest first).

## CN Status and Actions (3-status)
| Status | Meaning | Actions |
| --- | --- | --- |
| Draft | Form open, not confirmed | Edit · Delete (number retired) · Send (optional) |
| Applied to [invoice(s)] | Confirmed & applied | Cancel (full reversal) · Download PDF · Send to customer |
| Cancelled | Cancelled, effects reversed | Download PDF only |

## Notes / open
- Drafts appear in the list — **confirm with Design** whether shown by default or only under the Draft filter.

## ⚠️ Comparison vs current build (2026-07-13)
Our `CreditNotesList.tsx` predates the 719/818 status collapse — it's still built to the **old DES-763**
register. So the list is the least-aligned surface. Note: the user previously said "don't touch the credit
note list for now" — this analysis is the pre-work for whenever that's lifted.

**Gaps (build → 818):**
1. **Status model (root gap).** List uses 7 statuses — Open / Partially Applied / Fully Applied / Pending
   Refund / Partially Refunded / Refunded / Cancelled — plus an "Apply from list" step. 818 wants only
   **Draft / Applied / Cancelled**. Touches: `CNStatus` type, `STATUS_PILL`, `FILTERS`, seed `creditNotes.ts`,
   and the `applyFromList` / `saveFromList` derivation.
2. **Status filter chips** — 8 chips today; should be 4 (All / Draft / Applied / Cancelled).
3. **Date-range filter** — missing entirely (AC2 + Filter Options).
4. **Removable filter chips** — AC2 wants active filters as removable chips; we show a count badge on the
   Filters button + a checkbox list only.
5. **Issue date on the row** — a required column; our row shows customer / CN·invoice / amount / status, no date.
6. **Per-status actions from the list** — 818: Draft → Edit/Delete/Send · Applied → Cancel/Download/Send ·
   Cancelled → Download only. We wire Apply (Open) + Edit; no Delete (Draft), no Cancel (Applied) from the
   list. "Apply from list" is obsolete under 719 (Applied is immediate on Create).
7. **Draft CNs in the list** — no Draft seed rows, no Draft filter (818 open question: default vs filter-only).
8. **Sort: CN number** — we sort ascending; 818 wants descending. Minor.
9. **AC4 badge target** — CN badge exists on the invoice card (`onOpenCN`); confirm it lands on the CN detail
   (not the invoice's credits section).
10. **Refund statuses** — 818's enum has no refund states (refunds = DES-720/721). Our list folds refund CNs
    in via `refundState`. Decide whether refund CNs appear here as "Applied" or are excluded.

**Aligned:** separate screen from the invoice list; tap row → CN detail; customer filter; sort by issue
date (newest default) + amount (high→low); shared card/chip layout.
