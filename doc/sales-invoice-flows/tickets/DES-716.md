# DES-716 — Story 4 (CLIENT) — Client Upload Invoice

**Status:** In Progress · **Fetched:** 2026-07-14. Upload an existing invoice → OCR extract → review/edit
→ issue with the **user-entered** invoice number → status lifecycle.

## Acceptance Criteria
1. Upload PDF/JPG/PNG within size limit; reject unreadable/oversized/unsupported with a clear reason.
2. OCR → Draft pre-filled; low-confidence fields flagged; failure handling (nothing extracted → error/manual).
3. Issue with the **user-entered invoice number** (not system-generated); validate; → Awaiting Payment;
   duplicate number → warn + allow override.
4. Send optional — default **record-only**; may optionally email PDF / share link.
5. Cancel with Credit Note → **Void**.
6. Reconciliation status outcomes (Paid / Partially Paid / Overpaid) — **Block 3 scope, not tested here.**

## File Constraints
PDF / JPG / PNG · max 10 MB · reject unreadable/corrupt/unsupported at upload.

## Status Matrix
Draft (edit + delete) · Awaiting (limited: due date + reminder toggle) · Overdue (limited) · Partially
Paid · Paid · Void — all non-editable once past Draft/Awaiting.
