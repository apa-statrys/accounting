# DES-782 — Manage Recurring Sales Invoice · Story 48 (CLIENT)

_Golive: **feature must be OFF on Prod at launch.**_

## Goal
Let the client set up a recurring invoice series for a customer (schedule, amount, line items) so invoices
auto-generate each interval and appear in the Sales Invoice List, ready to send + reconcile.

> **KR (authoritative refinements, in blue on the ticket):**
> - Recurring should be **part of invoice creation** (a checkbox/toggle to tick), **not** a separate
>   "Recurring Invoices" section.
> - **Only the client** can set/pause/edit/cancel recurring series — **not the accountant/admin**.
> - Client **may edit the generated draft's fields before confirmation**.
> - Add an **Amount** field = Unit price × Quantity.

## Acceptance Criteria
1. **Create a series** — from invoice creation; setup form has all required fields; on confirm, series
   created + first invoice scheduled. _(The "preview of the next 3 scheduled dates" was struck out in the
   live ticket — no longer required.)_
2. **Auto-generate on each date** — new invoice with series line items; assigned `INV-YYYY-######`, status
   **Draft**; shows in list with a **Recurring badge**; client notified; **Auto-send on** → sent + Awaiting
   Payment; **off** → stays Draft for manual review/send.
3. **Confirm before issue** — generated Draft → client selects **Confirm Invoice** → status **Awaiting
   Payment** + journal (Dr AR / Cr Revenue) + ready to send. (Review step before commit.)
4. **Pause / resume / edit / cancel** — pause (no gen until resumed), resume, edit amount/frequency
   (**from next scheduled date only**), cancel (stops all future gen immediately; already-generated invoices
   remain and follow normal lifecycle).
5. **View full log** — series detail lists all generated invoices with **date, invoice number, amount, status
   (Generated / Skipped / Cancelled)**; navigate to any invoice; **next scheduled date shown at the top**.
6. **Auto-end** — on end date reached or max invoices hit → series **Completed**; no further gen; notify.

## Recurring Invoice Fields (all "Must be editable" per KR)
Customer* · Item Name/Description* · Unit Price* · Quantity* · Currency* (inherits client default) ·
Amount* (=UP×Qty) · Frequency* · Start Date* (future) · End Date · Max Invoices · Discount · Auto-send toggle.

## Frequency Options
Weekly (same weekday) · Bi-weekly (every 2w) · Monthly (same day) · Quarterly (same day, every 3m) · Annually.

## Series Status Rules
| Status | Meaning | Edit? | Actions |
|---|---|---|---|
| Active | Generating | Yes | Pause, Edit, Cancel |
| Paused | No gen until resumed | Yes | Resume, Edit, Cancel |
| Completed | End date / max-N reached | No | View log only |
| Cancelled | Stopped by client | No | View log only |
| Failed | System error (backend) | No | View log only — backend/out of scope |

## Edit Rules (post-creation)
Unit Price / Quantity / Frequency → editable, **from next scheduled date**. End date / max → editable
**immediately**. Customer / Currency / Start date → **NOT editable** (must cancel & create new series).

## Out of scope here (other stories / backend)
Sending, reconciliation, cancellation-with-CN (own stories). Admin App create/manage (**admin CANNOT**
create/pause/edit/cancel; accountant only picks COA). Journal/GL posting. Notifications.
