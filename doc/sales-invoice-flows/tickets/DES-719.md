## Background
Clients sometimes need to cancel or reduce an unpaid sales invoice (order cancelled, price corrected,
discount applied). A credit note is a formal document that reduces what the customer owes. It's raised
against an unpaid invoice for the same customer. **Once applied, a CN cannot be edited** — if raised in
error, the client cancels it entirely, which reverses all effects and restores the invoice.

## Goal
Let a client create a sales credit note against an unpaid invoice (Awaiting Payment / Overdue), apply it
to reduce or cancel the outstanding balance, and cancel it entirely if raised in error.

---

## Acceptance Criteria
1. **Open the New Credit Note form** — from the invoice list or an unpaid invoice; opens in **Draft**;
   system pre-fills the Pre-filled Fields; client completes the Client-Filled Fields before confirming.
2. **Confirm & create** — "Create Credit Note" validates all required fields; on pass the CN is created
   with status **Applied to [invoice number]**; CN number is system-generated (read-only); the linked
   invoice updates immediately per the Credit Outcome table; **the CN is not editable after this point**.
   *(Note: the live ticket labels this AC "2"; there is no AC3 — numbering skips.)*
4. **Cancel an applied credit note** — "Cancel Credit Note" + confirm → status **Cancelled**; linked
   invoice balance & status fully restored; reversing GL entry posted automatically; CN number retired
   (cannot be reused). A cancelled CN cannot be reinstated — open a New Credit Note to start again.
5. **Send the credit note** — email / shareable link / download, reusing the invoice send flow; email
   includes the CN document, invoice number(s), remaining balance payable, and due date; sending is
   optional (CN is applied regardless of whether it's sent).
6. **Create failure is recoverable** — clear error message; retains all entered details/allocations;
   correct and retry without re-entering data.
7. **Delete a draft Credit Note** — Draft only; confirmation prompt; permanently removes the CN; deletion
   available in no other status.

## Rules

### New Credit Note vs Create Credit Note
- **New Credit Note** (invoice list / invoice detail) → opens the blank CN form; nothing created yet.
- **Create Credit Note** (button at bottom of the form) → submits, creates, and immediately applies the
  CN to the linked invoice.

### Pre-filled Fields (editable before Create, except where noted)
| Field | Pre-filled from | Editable before Create? |
| --- | --- | --- |
| CN number | System — `CN-YYYY-######` (from 000001) | **No** — read-only always |
| Customer name | Selected customer record | **No** — not editable (simpler UX) |
| Associated invoice | Linked invoice | **No** — fixed |
| CN date | Today (default) | Yes — DD/MM/YYYY |
| Due date | Linked invoice due date | Yes — DD/MM/YYYY |
| CN currency | Linked invoice currency | **No** — fixed to invoice currency |
| Payment method | Statrys Primary VA (default) | **Yes** |
| Item Name | Linked invoice item name | **No** — fixed to invoice |

### Client-Filled Fields (all required)
| Field | Data type | Notes |
| --- | --- | --- |
| Reason | Enum | Return / Defect / Pricing error / Goodwill / Dispute |
| Description | String | Description of the credit being issued |
| Quantity | Decimal (2 dp) | Max 999,999.99 |
| Unit | Enum | unit / hour / day / month / session / pair (same as invoice line items) |
| Unit Price | Decimal (2 dp) | In CN currency |
| Amount (total CN) | Decimal (2 dp), calculated | Quantity × Unit Price — auto/read-only |
| Allocation per invoice | Decimal (2 dp) per linked invoice | Must sum to CN total (single-invoice: = CN total) |

### Credit Outcome
| Credited amount | Linked invoice result |
| --- | --- |
| = Full invoice amount | Invoice marked **Void** ("keep consistent terminology" = Cancelled) |
| < Full (partial) | Invoice stays **Awaiting Payment**; outstanding reduced by the credited amount |

### Carried-Over Data (pre-fill from invoice)
Client info (carried over, **not editable**), linked invoice number (stored as link, shown on CN), line
items / products & services (carried over; editable for a partial credit), currency (same as invoice,
not editable).

### Credit Note Status Matrix (3 states)
| Status | Meaning | Editable? | Cancellable? |
| --- | --- | --- | --- |
| Draft | Form open, not yet confirmed/applied | Yes — all fields except CN number | No — client can delete (record removed, number retired) |
| Applied to [invoice number] | CN confirmed & applied | **No** | Yes — full reversal |
| Cancelled | CN cancelled, all effects reversed | No | No |

### Validation Rules
Required fields complete → can create · required editable fields empty → not created (inform user) · CN
issue date invalid (must be DD/MM/YYYY) → not created · credited amount exceeds invoice outstanding →
not created (inform user).

## Notes
- **One CN per invoice in this flow.** If a CN already exists against this invoice, "New Credit Note" is
  not available — a second CN against the remaining balance goes through the multi-invoice flow **DES-821**.
- CN number `CN-YYYY-######` is assigned **when the form opens** (before confirmation); a deleted draft
  retires the number. Sequence independent of invoice numbering.
- Applies to **unpaid** invoices only (Awaiting / Overdue). For Paid invoices, the refund flow is DES-720/721.
- Journal reversal on issue is specified in the Chart of Accounts / Journal story, not here.
- **Out of scope:** the refund flow, the View Credit Note flow, journal posting.
- Reason field required: dropdown (return / defect / pricing error / goodwill / dispute) **plus optional
  free text** *(Notes only — the Client-Filled table lists Reason as plain Enum + a separate required
  Description; the "optional free text" line conflicts with the table).*
- **Go-live flag: this feature must be turned OFF on Prod at launch.**
- Surface: client side = mobile app; Operation Manager back office = web.

## Things Not Clear (open)
