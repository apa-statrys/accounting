# [DES-817] Edit Sales Invoice Fields

## Background

A sales invoice is the primary billing document issued by the client to their customers or vendors.
Fields are drawn from three sources: account settings (company info, logo), system generation
(number, status, type, source), and client input at creation time. Edit rights on each field depend
on the invoice's current status — some fields are always read-only, others are editable only in
Draft, and nothing is editable once the invoice is Paid.

## Goal

Define all fields on a sales invoice, their source, data type, and editability by status, so that
the client can edit each invoice before sending and the system enforces the right restrictions at
every stage of the lifecycle.

---

## Acceptance Criteria

1. **Auto-populated fields pre-fill on invoice creation**
   GIVEN a client creates a new sales invoice
   WHEN the invoice form opens
   THEN all fields marked Auto-populated in the Invoice Fields table pre-fill without client input
   AND system-generated fields (invoice number, status, type, source) are read-only and cannot be edited

2. **Client edits all editable fields in Draft**
   GIVEN the invoice is in Draft status
   WHEN the client fills in the pre-filled content
   THEN they can edit all fields marked Editable in Draft in the Invoice Fields table
   AND changes to line-item Quantity or Unit Price are reflected immediately in the calculated
   Amount, Subtotal, and Total

3. **Client manages line items in Draft**
   GIVEN the invoice is in Draft status
   WHEN the client interacts with the line-item section
   THEN they can add new lines (Add Line) and remove existing lines (Remove Line)
   AND each line Amount = Quantity × Unit Price (auto)
   AND at least one line item must exist before the invoice can be sent

4. **Edit restrictions apply after Send**
   GIVEN the invoice is in Awaiting Payment or Overdue status
   THEN only fields marked Editable after Send remain editable
   AND all other fields are locked / read-only
   AND line items cannot be added, edited, or removed after Send

5. **No fields are editable after Paid**
   GIVEN the invoice is in Paid, Partially Paid, or Void status
   THEN no field is editable, no line item can be changed, and the invoice can only be viewed

6. **Client controls the automated reminder per invoice**
   GIVEN an invoice in Draft, Awaiting Payment, or Overdue status
   THEN an Automated reminder toggle is displayed
   AND it defaults to the account-level reminder setting (Invoice Settings)
   AND the client can turn it on/off for this invoice independently of the account-level setting
   AND once the invoice moves to Paid, the reminder auto-deactivates and the toggle is no longer shown

## Rules

### Invoice Field Table

| Field | Source | Data type | Required | Editable in Draft | Editable after Send | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| Company Information | Auto — Invoice Settings | String | Yes | No | No | |
| Company Logo | Auto — Invoice Settings | Image (JPG/JPEG/PNG, max 10 MB) | No | No | No | |
| Client (Customer) | User entry | String — selected from saved client records | Yes | **No** | No | |
| Client Email Address | Auto — from selected client record | String (valid email) | Yes | No | No | |
| Invoice Number | System-generated | String — INV-YYYY-###### (from 000001) | Yes | n/a | No | Generated after submission |
| Currency | Auto — Invoice Settings default | Enum — dropdown | Yes | **Yes** | No | |
| Issue Date | User entry | Date — DD/MM/YYYY | Yes | **Yes** | No | |
| Due Date | User entry | Date — DD/MM/YYYY or preset (30/60/90/Custom) | Yes | **Yes** | **Yes** | |
| Receiving Account | User entry | Enum — dropdown | Yes | **Yes** | No | Default Statrys Primary VA. Options: Statrys Primary VA / Other Statrys VA / Other registered bank accounts |
| Invoice Type | System-generated | Enum — Sales / Purchases | Yes | No | No | |
| Invoice Source | System-generated | Enum — Created / Uploaded | Yes | No | No | "Created" = manual; "Uploaded" = upload flow |
| Invoice Status | System-generated | Enum — Draft / Awaiting Payment / Overdue / Partially Paid / Paid / Void | Yes | No | No | Event-based |
| Automated Reminder | User entry | Boolean toggle | Yes | **Yes** | **Yes** | |

### Invoice Line Fields

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| Item Name | String | Yes | Item title |
| Description | String | **Yes** | Description of the sales item |
| Quantity | Decimal (2 dp) | Yes | Max 999,999 (pending Design) |
| Unit | Enum — unit / hour / day / month / session / pair | Yes | Dropdown, fixed list |
| Unit Price | Decimal (2 dp) | Yes | In invoice currency |
| Amount | Decimal (2 dp) | Yes | Quantity × Unit Price |
| Remove Line / Add Line | Button | — | Manage line items |
| Automated reminder | Toggle | Yes | Per invoice |

### Discount & Summary

| Field | Type | Notes |
| --- | --- | --- |
| Discount per invoice | Decimal (2 dp) or percentage | Optional |
| Subtotal | System (2 dp) | Sum of line amounts |
| Total Discount | System (2 dp) | Shown only if a discount is entered |
| Total | System (2 dp) | Total after discount |

## Notes

- Company info and logo are set once in account settings and applied to all invoices; not editable per invoice.
- Customer address/contact auto-fill from the selected client record. To change it, edit the client
  record — not within the single invoice.

## Things Not Clear (open)

- **Receiving Account options scope:** dropdown = Statrys Primary VA (default) / Other Statrys VA /
  other registered bank accounts. Confirm with Tech which accounts count as "other registered bank
  accounts". Button is hidden in design until functionality is fully developed — confirm phasing
  with Design & Tech.

## Decisions & conflicts vs current build

- **Client (Customer) locked in Draft — DECIDED 2026-07-09 (lock per ticket).** The customer is fixed
  once chosen at creation. When editing an existing invoice (`isEditing`) the customer card is now
  read-only (no tap-to-change); the tappable "Change customer" affordance remains only in the fresh
  create flow. To change it, start a new invoice / edit the client record.
- **Line Description Required — DECIDED 2026-07-09 (required per ticket).** Re-added the `*` on
  Description in `AddServicesSheet` and gated "Add item" on it (reverting the DES-715-era optional
  treatment).
- **Editable-after-Send = Due Date + Automated Reminder only — DONE 2026-07-09 (Figma 1130-6193).**
  The issued limited-edit screen (Awaiting/Overdue) now locks customer, Issue Date, Currency,
  Receiving Account, line items (read-only, no add/remove/swipe), and Discount (toggle disabled);
  only Due Date (chevron) and the Automatic reminders toggle stay editable. This replaces the older
  broader limited-edit set (customer/items/receiving account/discount) documented in CLAUDE.md.
